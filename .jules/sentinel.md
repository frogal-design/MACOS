## 2026-06-25 - [Build-Time Environment Variable Exposure in Client Bundles]
**Vulnerability:** Exposing build-time configuration (specifically `process.env.GEMINI_API_KEY`) via Vite's `define` property. When a React application is compiled for the client-side, any value defined under `define` or prefixed with `VITE_` is statically baked/inlined into the production JavaScript bundle, making the secret visible to anyone inspecting the network or source code.
**Learning:** The application was configured to load and define `GEMINI_API_KEY` globally even though no actual Gemini AI integration, API calls, or server-side actions were used in the source code. This redundant configuration unnecessarily introduced a severe credential exposure hazard during production compilation.
**Prevention:** Never use global substitution/defining (`define` or `process.env`) for secrets unless explicitly and securely required. In a purely frontend/client-side single page application (SPA), do not bake credentials into the build artifacts; always route API requests through a secure server-side proxy or backend API gateway where credentials can be stored and used securely at runtime. Remove unused credentials, API client packages, and redundant environment loaders (`dotenv`, `@google/genai`) to minimize attack surface and dependency supply-chain risks.
