## 2026-07-22 - [Supply Chain Surface Reduction & Direct Dependency Hardening]
**Vulnerability:** Unused dependencies (express, @google/genai, dotenv) and vulnerable packages (react-router-dom, vite) in `package.json`, along with potential secret exposure in `vite.config.ts` via the `define` property.
**Learning:** Having unused server-side packages like `express` in a client-side only Vite React application unnecessarily expands the supply chain attack surface and introduces vulnerabilities from transitive dependencies (e.g., `qs`). Furthermore, using Vite's `define` property to inject environment variables (like `GEMINI_API_KEY`) globally can lead to secret leakages in compiled assets, even if the application does not actively reference those variables in source code.
**Prevention:** Remove unused dependencies and keep critical build tools and router libraries updated. Avoid using the global `define` compiler replacement for environment variables unless they are explicitly required in client-side code, and favor Vite's native `import.meta.env` system which only compiles variables explicitly prefixed or referenced in the source.
