## 2026-07-23 - Supply Chain Surface Reduction & Direct Dependency Hardening
**Vulnerability:** Insecure API key leakage risk (GEMINI_API_KEY) was present in `vite.config.ts` via the `define` property, and multiple high-severity dependencies like `react-router-dom` and `vite` were outdated, alongside unused backend/utility dependencies (`@google/genai`, `express`, `@types/express`, `dotenv`) introducing unnecessary supply chain attack surface.
**Learning:** Hardcoding or injecting secrets in client-side bundling configurations exposes those keys directly to production JavaScript assets, which can be easily extracted. Additionally, maintaining unused direct dependencies increases vulnerability exposure and dependency graph complexity.
**Prevention:** Avoid injecting server-side secrets into Vite's `define` config for frontend bundles, utilize standard `.env` variables via `import.meta.env` dynamically if needed (with `VITE_` prefix), regularly execute `npm audit` to identify and update vulnerable direct dependencies, and systematically uninstall any unused packages.
