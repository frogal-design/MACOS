## 2026-03-24 - [CRITICAL] GEMINI_API_KEY Exposure in Vite Configuration
**Vulnerability:** The `GEMINI_API_KEY` was being injected into the client-side bundle via Vite's `define` property in `vite.config.ts`. This made the secret accessible to any user visiting the application.
**Learning:** Vite's `define` property performs a global search-and-replace during build time. Even if the variable is not explicitly used in the source code, defining it can expose it if the replacement pattern is broad or if it's accidentally referenced. Additionally, relying on `loadEnv` in `vite.config.ts` to inject secrets into `define` is a high-risk pattern for frontend applications.
**Prevention:** Never use Vite's `define` to inject sensitive environment variables into the client bundle. All AI API calls or sensitive operations must be handled on the server side. Add explicit 🛡️ security warnings to `.env.example` and `README.md` to educate developers and prevent future regressions.
