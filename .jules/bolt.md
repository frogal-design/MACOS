## 2026-06-22 - [Gallery Optimization: Hoisting and State Simplification]
**Learning:** React components that define and sort static data within their body incur O(n log n) overhead on every render. Additionally, tracking both a selected object and its index in state is redundant and error-prone.
**Action:** Always hoist static data arrays to module scope and pre-sort them if the order is fixed. Refactor modal state to use a single source of truth (e.g., `selectedIndex`) and derive other values to ensure consistency and minimize state updates.
