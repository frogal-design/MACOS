## 2026-06-18 - Optimized Data Handling and DOM Density
**Learning:** In text-heavy React applications, components that animate individual characters (like `DecryptedText`) can create significant DOM density if they leave hundreds of `<span>` elements in the tree after the animation is complete. This increases memory overhead and can slow down subsequent style recalculations. Additionally, performing $O(n \log n)$ operations (like `.sort()`) or re-allocating static arrays inside the render cycle adds unnecessary overhead to every frame.
**Action:** Always flatten animated text components into a single text node once the animation reaches a terminal state. Hoist static data arrays to module scope and pre-sort them at the top level to ensure the render cycle remains $O(n)$ or better.
